# 🚀 COMPLETE DEVOPS GUIDE: ARTIFACTS, DOCKER & PYPI (END-TO-END)

# 🧠 1. BASIC CONCEPT

DevOps mein har language ka "build output" hota hai:

Java → .jar (compiled application)
Python → .whl (packaged module)
Node.js → dist/ (production build output)

👉 Ye sab artifacts kehlate hain

---

# 🧱 2. DOCKER IMAGE BASIC RULE

Docker image banane ka formula:

Base Image + Runtime + Build Artifact + Run Command

---

# ☕ 3. JAVA (.jar) → DOCKER IMAGE

📦 Artifact:
target/app.jar

🐳 Dockerfile:

FROM openjdk:17
WORKDIR /app
COPY target/app.jar app.jar
CMD ["java", "-jar", "app.jar"]

🔨 Build:
docker build -t java-app .

▶ Run:
docker run -p 8080:8080 java-app

---

# 🐍 4. PYTHON (.whl) → DOCKER IMAGE

📦 Artifact:
dist/myapp.whl

🐳 Dockerfile:

FROM python:3.10
WORKDIR /app
COPY dist/*.whl .
RUN pip install *.whl
CMD ["python", "-m", "myapp"]

🔨 Build:
docker build -t python-app .

▶ Run:
docker run python-app

---

# 🌐 5. NODE.JS (dist/) → DOCKER IMAGE

📦 Artifact:
dist/

🐳 Frontend Dockerfile:

FROM nginx:alpine
COPY dist/ /usr/share/nginx/html

🔨 Build:
docker build -t frontend-app .

▶ Run:
docker run -p 80:80 frontend-app

---

# 🧠 6. BEST PRACTICE

❌ Bad:
COPY . .

✅ Good:
COPY only build artifact

👉 Reason:
- fast build
- secure image
- less size

---

# 📦 7. PYTHON PACKAGE BUILD (.whl)

📁 setup.py:

from setuptools import setup, find_packages

setup(
    name="myapp",
    version="0.1",
    packages=find_packages(),
)

---

# 🔨 BUILD PACKAGE

pip install build
python -m build

👉 Output:
dist/
 ├── myapp-0.1.whl
 ├── myapp-0.1.tar.gz

# 🚀 8. PYPI UPLOAD PROCESS

📦 Step 1: Install twine

pip install twine

---

📦 Step 2: Create PyPI account
https://pypi.org

---

📦 Step 3: Upload package

twine upload dist/*

---

📦 Step 4: Install from PyPI

pip install myapp

---

# 🔐 9. SECURE UPLOAD (BEST PRACTICE)

twine upload dist/* -u __token__ -p YOUR_API_TOKEN

---

# 🔄 10. FULL DEVOPS FLOW

Code
↓
Build Artifact (.jar / .whl / dist)
↓
Docker Image Build
↓
Push to Registry (DockerHub / ECR)
↓
Deploy (Kubernetes / Server)

---

# 🎯 11. INTERVIEW ANSWER

A build artifact like .jar, .whl, or dist is first generated from source code.
Then it is packaged into a Docker image using a base runtime like OpenJDK, Python, or Nginx.
For Python, the .whl package can also be uploaded to PyPI using tools like build and twine for distribution.

---

# 🧠 12. ONE LINE MEMORY

.jar = Java compiled app
.whl = Python package
.dist = Node build output

Docker = runtime packaging
PyPI = Python distribution store
